Project Dragon: System Overview and Core Technologies

Project Dragon is a next-generation logistics management platform designed to optimize supply chain operations through real-time data analysis and predictive modeling. The core of the system is a microservices architecture, where different functionalities are handled by independent, containerized services. The primary backend language for all services is Python 3.11, chosen for its robust ecosystem of data science libraries and its rapid development capabilities.

The primary user-facing component is the main dashboard, which provides a comprehensive view of all ongoing shipments, inventory levels, and carrier performance metrics. This dashboard is a single-page application built using the React framework. It communicates with the backend services via a REST API, which is secured using JWT authentication. The standard SLA (Service Level Agreement) for API response time is under 200 milliseconds.

Data ingestion is handled by a dedicated pipeline that processes shipment manifests, which are typically received as CSV files via an SFTP server. Once a new file is detected, a serverless function is triggered. This function validates the data for correctness, cleans the records, and then loads the structured information into a PostgreSQL database. The database schema is highly normalized to ensure data integrity and prevent anomalies.
